## Why

The current server pairs a FastAPI process with a `whisper.cpp` `whisper-server` subprocess that we manage ourselves (~188 lines of lifecycle code, auto-restart, port wiring). Every recent bug fix added more code to that manager, the Docker build takes 10â€“15 minutes compiling whisper.cpp, and we cannot add a real streaming endpoint without bringing in a second backend (CTranslate2). On top of that, the Apple Shortcut UX has only one round-trip and no streaming feedback, so a user has no way to see partial transcripts or pose follow-up questions without a separate web client. This change collapses the architecture to a single in-process backend (`faster-whisper` + CTranslate2 `int8_float16`), adds the missing endpoints needed for the upcoming PWA front-end, and unifies the deployment story across a Mac mini and a future PVE + RTX 3070 Ti host. See `docs/PRD-roadmap.md` for the full motivation.

## What Changes

- **BREAKING** Remove the `whisper.cpp` git submodule and the `whisper-server` subprocess; load CTranslate2 models in-process via `faster-whisper`. Removes `app/services/whisper_manager.py`, rewrites `app/services/whisper.py` from an HTTP client to an in-process wrapper, and removes the `WHISPER_SERVER_HOST`/`WHISPER_SERVER_PORT`/`WHISPER_SERVER_URL`/`WHISPER_AUTO_RESTART`/`WHISPER_BINARY_PATH`/`WHISPER_MAX_RETRIES`/`MODEL_PATH` env vars.
- **BREAKING** Merge `POST /transcribe-raw` into `POST /transcribe` using Content-Type dispatch (`multipart/form-data` keeps the existing form behaviour; `audio/*` or `application/octet-stream` is read directly as raw audio body). The `language` query parameter continues to work on both shapes.
- **BREAKING** Rename `GET /health` to `GET /status` and expand the response to include active model name/path, `compute_type`, `device`, `loaded` flag, load time, server version, uptime, and Gemini configuration state.
- **BREAKING** `registry/models.yaml` schema changes from GGML single-file entries to CTranslate2 directory entries with a reserved `format` discriminator (only `"ct2"` is accepted in v2.0; other values are rejected with a clear error so the field remains forward-compatible for future formats). The default model becomes `breeze-asr-25` (Breeze ASR 25 in CTranslate2 `int8_float16`) and `large-v3-turbo` is retained as the multilingual fallback. The previous GGML-only built-in entries (`large-v3-turbo-q8`, `large-v3`, `medium`, `base`, and the GGML `breeze-asr-25` entry) are removed from the shipped registry; users who depended on those entries SHALL add equivalent CT2 entries (for example `Systran/faster-whisper-medium`) to their own registry copy after upgrading. `scripts/model-manager.sh` is rewritten (~13 KB â†’ ~3â€“4 KB) to download via `huggingface-cli`; the `make models`, `make download-model`, `make set-model`, and `make delete-model` user interface is preserved.
- Add `POST /ask`: accepts the same audio inputs as `/transcribe` (with the same `language` and `prompt` query parameters and identical post-processing) plus `application/json {"text": "..."}`. Runs the configured Gemini model (`gemini-2.5-flash` default) and returns `{"transcript": <string or null>, "answer": "..."}` (transcript is `null` for the JSON text path). Supports `?stream=true` for Server-Sent Events streaming where every `data:` line is a single JSON document; the event sequence is one `transcript` event, then zero or more `token` events (zero only on the empty-LLM-completion edge case), then a terminating `done` event on success or a terminating `error` event on failure. System prompt is configurable via a new `GEMINI_SYSTEM_PROMPT` env var alongside `GEMINI_API_KEY` and `GEMINI_MODEL`; if `GEMINI_SYSTEM_PROMPT` is unset, a baked-in default Taiwan-friendly assistant persona is used.
- Add `WS /listen`: bidirectional WebSocket accepting 16 kHz mono `pcm_s16le` binary frames (recommended ~250 ms per frame; accepted frame size range 200 bytes to 64 KiB inclusive) and emitting JSON text frames `{"type":"partial","text":...,"start_ms":...,"end_ms":...}` and `{"type":"final",...}`. A single connection MAY produce multiple utterances; closing the socket mid-utterance discards the in-flight buffer without emitting a `final`. Endpointing uses a server-side VAD plus a sliding-window wrapper around the shared in-process `WhisperModel` instance (the wrapper is implemented inside `app/services/stream.py`, not depended on as an external package â€” see design.md for rationale).
- Update the bundled Apple Shortcut template: replace the `/transcribe-raw` URL with `/transcribe`, add an `Ask` variant pointing at `/ask`, add `Vibrate Device` before the request and `Show Notification` after, and re-shoot `docs/ios-shortcuts-workflow.jpeg` so the documentation reflects v2.0 flows. The screenshot refresh is non-blocking for the v2.0.0 tag â€” it is tracked as a separate task that MAY land in a post-tag follow-up.
- Add a `GET /` API discovery response that lists every registered endpoint with its HTTP verb and one-line description (replacing the v1 discovery payload).
- Add new dependencies: `faster-whisper` and `google-genai`. WebSocket and SSE transport rely on the existing `uvicorn[standard]` and FastAPI `StreamingResponse`; no separate streaming package is added.

## Capabilities

### New Capabilities

- `ask`: Voice/text question-answering endpoint that combines server-side transcription with a configurable Gemini model and supports both blocking and SSE streaming response modes.
- `transcribe-stream`: WebSocket endpoint that consumes PCM audio chunks and emits incremental partial/final transcripts with millisecond timestamps for live captioning use cases.
- `status`: Detailed health endpoint that reports the loaded ASR model, runtime device, compute type, server version, uptime, and LLM configuration so operators can distinguish multiple deployments (Mac mini vs GPU server) at a glance.

### Modified Capabilities

- `inference-params`: The endpoint surface changes â€” `POST /transcribe-raw` is removed and its raw-binary behaviour is reachable via `POST /transcribe` through Content-Type dispatch; the `language` parameter requirement is restated against the unified endpoint.
- `model-management`: Existing `make models / download-model / set-model / delete-model` commands keep their semantics but operate on CTranslate2 model directories and use `huggingface-cli` to fetch the Breeze ASR 25 CT2 repository.
- `model-registry`: The `registry/models.yaml` schema changes â€” required fields become `repo_id`, `format` (only `"ct2"` accepted in v2.0; the field is a reserved discriminator for future formats), `compute_type`, `local_dir`, `size`, `languages`, and `description`. Optional fields `subfolder` (for Hugging Face repositories that publish multiple quantisations under one repo) and `revision` (commit SHA or branch name for reproducible pins) are added. The previous GGML-only fields (`url`, `filename`) are removed.
